# ToDoアプリ ナチュラルテーマ デザインガイドライン

## デザイン原則
- **やさしい自然感覚**: 柔らかな色調と余白で呼吸感をつくり、乱雑さを避ける。
- **明瞭さと階層構造**: 重要度が高い要素は彩度とサイズで強調し、Apple HIG の「Clarity」を意識する。
- **素材感のある深度**: 影・レイヤリングは控えめにしながらも浮遊感を持たせ、「Depth」を演出する。
- **適応性の高いレイアウト**: レスポンシブに動く8ptグリッドを基準にし、ダイナミックタイプ・アクセシビリティに対応する。

## カラーシステム
| 役割 | カラー | HEX | 用途 |
| --- | --- | --- | --- |
| プライマリ | リーフグリーン | `#6BB892` | 主要アクションボタン、進行状況のインジケーター |
| セカンダリ | サンドベージュ | `#EFDCC5` | サブアクション、強調背景、セクション分割 |
| アクセント | スカイブルー | `#82C7C9` | 情報の補足、リンク、タグ |
| サーフェス | アルバスター | `#F8F5F0` | 基本背景、カード内側 |
| ベース | ウォームグレイ | `#E5DED3` | アプリ全体の背景、セクションラップ |
| テキスト(プライマリ) | スレートブラウン | `#3E3A36` | 本文、見出し |
| テキスト(セカンダリ) | モスグレイ | `#6B665F` | 補足テキスト、ラベル |
| ディスエーブルド | ミストグレイ | `#B8B3AB` | 無効状態のボーダー・テキスト |
| 成功 | フォレストグリーン | `#4E8F6F` | 達成済みタスク、成功メッセージ |
| 警告 | ゴールデンアンバー | `#D6A75B` | 期限間近アラート |
| エラー | ローズブリック | `#C97171` | エラー通知、重要な注意喚起 |

- ライトモード前提。ダークモード対応時はサーフェスとベースを `#1F1B17` 系に反転し、彩度とコントラストを維持する。
- グラデーション使用時はプライマリ+アクセントの15°～30°角度を推奨し、淡いトーンに留める。

## タイポグラフィ
- **基本フォント**: Appleプラットフォームに倣い `SF Pro`（和文は `SF Pro JP`）を使用する。Web利用時は `Noto Sans JP` をフォールバックに設定。
- **見出しスケール（Dynamic Type対応）**  
  - H1: 28pt / Regular（Bold不可）。タスク一覧画面のタイトルに使用。  
  - H2: 22pt / Medium。セクション見出し。  
  - H3: 18pt / Medium。カード内タイトル。  
  - Body Large: 16pt / Regular。主要本文。  
  - Body Small: 14pt / Regular。補足情報。  
  - Caption: 12pt / Regular。ラベル・タイムスタンプ。  
- **行間**: 各スタイルのフォントサイズ×1.4。長文は1.6まで拡張可。
- **文字間**: H1のみ -2%、その他はデフォルト。大文字表記は避け、タイトルケースを意識する。

## 余白・間隔
- 基本ユニットは8pt。主要要素間は16pt、セクション間は24pt。密度調整が必要な場合は4pt単位で補正。
- カード内パディング: 上下16pt、左右20pt。ボタン内パディング: 垂直12pt/水平20pt。
- アイコンと文字の間隔は8pt。ラベルやタグの水平ギャップは6ptでコンパクトさを維持。
- タスクリストの行間隔は12pt＋境界に1ptの仕切り線（`#E7E1D9`）。

## 角丸
- ボタン: 16pt（プライマリ）、12pt（セカンダリ・ミニ）。角は一定に保ち、角丸値はグリッドに合わせて変更。
- カード: 20pt。カード同士が重なる場合は一段浅いレイヤーを16ptに設定し階層差を示す。
- 入力フィールド・バッジ: 12pt。チェックボックスは16ptの円形スイッチで統一。

## 影の効果
- 基本影: `0 8 24 0` (X Y Blur Spread) + `rgba(107, 102, 95, 0.12)`。カード・モーダルに適用。
- 浮上レベル2: `0 16 32 0` + `rgba(62, 58, 54, 0.16)`。ダイアログ・メニュー用。
- 内側グロー: `inset 0 0 0 1px rgba(255, 255, 255, 0.65)` をサーフェス上で使用し、柔らかい光沢を演出。
- 影は背景に合わせて彩度を調整し、ダークモードはアルファを0.24まで増やす。

## コンポーネント設計
- **プライマリボタン**: プライマリカラー塗りつぶし＋ホワイトテキスト。ホバー時は彩度+8%、押下時は内側影でフィードバック。
- **セカンダリボタン**: 1px ボーダー（`#6BB892`）、背景は透明。ホバーでサンドベージュを敷き、テキスト色はプライマリ。
- **カード**: タスクカードは2カラム（左: ステータス、右: 詳細）で構成。左カラム幅72pt。右側にタスク名、期限、タグ、操作ボタンを縦整列。
- **チェックインジケーター**: 円形スイッチ。未完了は空色と1pxボーダー、完了時にはプライマリカラーで塗り潰し、チェックマークは `#F8F5F0`。
- **タグ・バッジ**: アクセントカラー背景、テキストはモスグレイ。8pt角丸、水平8pt/垂直4ptパディング。
- **モーダル**: 横幅 320〜480pt、外側余白32pt確保。上部にH2と補足テキスト、中央にフォーム、下部にボタンを水平配置。
- **リストセル**: 1pt区切り線、セル上下12ptパディング。セルタップ時にはサンドベージュでハイライト。

## アクセシビリティ配慮
- コントラスト比は最低でも 4.5:1。プライマリボタン（`#6BB892`）とホワイト文字は達成済み、セカンダリ背景を使用する際はテキストをモスグレイ以上のコントラストにする。
- Dynamic Type・文字サイズ変更に追従し、コンポーネントの高さとレイアウトは最小44pt（タップ対象）を確保。
- VoiceOver を意識し、ボタン・チェックなどインタラクティブ要素には動詞＋コンテキストのラベルを設ける（例: 「完了にする、今日のタスク」）。
- 色覚多様性に配慮し、状態表示は色だけに頼らずアイコンやラベルで補完する。
- アニメーションは200ms以内で緩やかな`ease-out`。ユーザー設定の「動きを減らす」がオンの場合はフェードのみに切替える。
- タッチジェスチャーの説明やデフォルト操作を Apple HIG に沿って提供し、プラットフォームの整合性を保つ。

## 運用のヒント
- HIG の更新を四半期ごとに確認し、コンポーネントのインタラクションや余白ルールを最新に保つ。
- デザインツールでは8ptグリッドとレイヤースタイルをライブラリ化し、チームで再利用しやすくする。
- 開発との連携時は `Figma Tokens` 等でカラーとタイポグラフィを変数化し、実装時の乖離を防ぐ。
