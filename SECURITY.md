# セキュリティポリシー

本プロジェクトでは、`AGENTS.md` に定義された運用ルールに従い、セキュリティを最優先で扱います。

## 脆弱性の報告方法
- コアチーム宛てに `security@test-codex.local` へメールを送ってください（実運用の連絡先に差し替えてください）。
- 問題の概要、再現手順、影響範囲、想定される被害をできる限り詳しく記載してください。
- 1 営業日以内に受付連絡を行い、3 営業日以内にトリアージ結果をご連絡します。

セキュリティ脆弱性については **公開 Issue を作成しない** でください。

## 協調開示
- 対応策が準備でき次第、開示タイミングを報告者と調整します。
- 必要に応じて過去バージョンにも修正を適用し、`vMAJOR.MINOR.PATCH` 形式のタグ付きリリースで配布します。

## サプライチェーンと依存関係
- Dependabot を週次で実行し、ルート npm・`apps/web`・GitHub Actions の更新をモニタリングします。作成された PR は迅速に確認してマージまたはクローズしてください。
- 依存関係に関わる高リスク修正を出荷する前に `npm audit` を実行します。

## シークレット管理
- すべてのシークレットは GitHub Actions Secrets か環境専用のキーバンクに保存し、ソースコードや `.env`（本番情報）に含めないでください。
- 万一シークレットを誤ってコミットした場合は、即座にキーをローテーションし、履歴のクリーンアップを依頼してください。

## セキュリティツール チェックリスト
以下の GitHub 機能を常に有効にしてください。
- Secret scanning および Push protection
- Dependabot alerts / Dependabot security updates
- CodeQL code scanning
- 必須ステータスチェック付きで保護された `main` ブランチ

2FA（2 段階認証）は強く推奨しますが、現在のポリシーでは必須ではありません。
